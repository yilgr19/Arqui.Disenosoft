Código del Proyecto

El proyecto incluye una aplicación web sencilla dentro de la carpeta principal.

Archivos principales

index.html – Página principal de la aplicación

style.css – Hoja de estilos

app-legacy.js – Lógica original del proyecto antes de refactorizar

Estos archivos se utilizan para:

Visualizar los problemas originales del código

Aplicar estandarización

Realizar depuración y mejoras

Evaluar métricas de calidad y mantenibilidad

Objetivo del Proyecto

El propósito principal es analizar, refactorizar, estandarizar y mejorar un código legacy, documentando cada etapa mediante auditorías de calidad y reportes estructurados.

Incluye:

Identificación de problemas técnicos y de estilo

Aplicación de estándares (nomenclatura, formato, buenas prácticas)

Refactorización para mejorar legibilidad y mantenibilidad

Evidencia comparativa del antes y después

Documentación soporte en español e inglés
